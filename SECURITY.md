# Security Policy

## Область применения

**Штрафинатор 2000** — внутренний инструмент компании CloudPayments для автоматизации подготовки документов. Данный документ предназначен для служб информационной безопасности при оценке рисков применения программы.

---

## Архитектура и поверхность атаки

Приложение является **изолированным десктопным инструментом** без каких-либо сетевых функций.

| Характеристика | Значение |
|---|---|
| Тип приложения | Десктопное (локальное), без сервера |
| Сетевые подключения | **Отсутствуют** |
| Внешние API / сторонние сервисы | **Не используются** |
| Хранение данных | **Нет** — данные не сохраняются и не кэшируются за пределами рабочей сессии |
| Передача данных | **Нет** — данные не покидают рабочую станцию пользователя |
| Аутентификация / авторизация | **Не требуется** — нет пользовательских аккаунтов и паролей |
| Привилегии ОС | Стандартные права пользователя, без запроса прав администратора |

---

## Что делает приложение

1. Читает файл `.xlsx` с локального диска, предоставленный пользователем.
2. Формирует на основе данных из файла документы `.docx`.
3. Сохраняет документы `.docx` в указанную пользователем папку на локальном диске.
4. Завершает работу.

На этом весь цикл работы программы заканчивается.

---

## Что приложение не делает

- Не устанавливает сетевые соединения (исходящих и входящих запросов нет).
- Не обращается к базам данных, платёжным системам или банковской инфраструктуре.
- Не читает и не изменяет системные файлы и реестр ОС.
- Не запрашивает повышенных привилегий.
- Не собирает телеметрию и не отправляет аналитику.
- Не устанавливает фоновые службы или автозагрузку.

---

## Зависимости

Приложение использует исключительно стандартные библиотеки Python с открытым исходным кодом:

| Библиотека | Назначение |
|---|---|
| `openpyxl` | Чтение `.xlsx` файлов |
| `python-docx` | Формирование `.docx` файлов |
| `customtkinter` | Графический интерфейс (окно приложения) |

Все зависимости встроены в исполняемый файл и не требуют установки стороннего ПО.

---

## Оценка остаточных рисков

**Единственный теоретически возможный вектор** — намеренно сформированный вредоносный `.xlsx` файл, эксплуатирующий уязвимость в библиотеке `openpyxl`. Данный риск:

- Является риском **цепочки поставок** библиотеки `openpyxl`, а не самого приложения.
- Не позволяет атакующему получить доступ к банковской инфраструктуре, так как у приложения нет соединения с ней.
- Актуален **только** при условии, что злоумышленник уже имеет физический или логический доступ к рабочей станции пользователя — что само по себе является инцидентом более высокого уровня.

**Итоговый уровень риска для банковской инфраструктуры: минимальный.**
